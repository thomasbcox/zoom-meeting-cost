## Findings (most severe first)

1. **[SAST · medium] GCM decryption without an explicit `authTagLength`** — 2 sites:
   `server/src/zoom/appContext.js:45` (decrypts the external Zoom app context) and
   `server/src/store/rateCrypto.js:56` (at-rest store envelope). Semgrep rule
   `gcm-no-tag-length`. Without pinning `{ authTagLength: 16 }`, GCM can be tricked into
   accepting a truncated auth tag, easing ciphertext forgery. The `appContext` site is the
   exposed one (attacker-supplied input, and the 401/200 response is a padding-oracle-style
   signal), gated by the client secret; `rateCrypto` needs volume write access.
   **Mitigation already in place:** Node ≥22 enforces stricter tag-length checks, which
   lowers practical severity — but pinning the length is unambiguous best practice on an
   auth boundary. **Fix:** pass `{ authTagLength: 16 }` to both `createDecipheriv` calls.
2. **[CI gap · medium] CI runs the test+build gate only** (`.github/workflows/ci.yml`),
   **plus CodeQL SAST** (`codeql.yml`) — but **no lint/format and no `shellcheck`/`shfmt`**.
   Per Thomas's estate-wide standard, `shellcheck` + `shfmt -d -i 2 -ci` **belong in CI**;
   they are absent here. There is also no `eslint`/`prettier` config at all for the JS.
3. **[Shell · low] `dev-docs/marketing/marketplace-cover.sh`** fails `shfmt -i 2 -ci`
   (compact one-liners not split; continuation indent) and has 2 `SC2016` **info** notes
   (single-quoted `$1,240` / `$18/min` are intentional ImageMagick literals — benign). The
   `.githooks/pre-commit` hook is `shfmt`- and `shellcheck`-clean.
4. **[Secret scan · false positive] `gitleaks` reported 3 hits, all synthetic test
   fixtures** (values redacted, never persisted):
   - `generic-api-key` · `scripts/secret-scan/detect.test.mjs:13` — the secret-scanner's own detection test.
   - `generic-api-key` · `server/test/loadEnv.test.js:10` — a synthetic env fixture.
   - `private-key` · `reviews/secret-scan-guardrails.codex.json:1` — a fake PEM in a review artifact.
   These are intentional fake secrets (repo policy: synthetic fixtures only) that `gitleaks`
   can't distinguish from real ones. **Not a leak.** Remediation is noise-reduction only:
   add a `.gitleaksignore` for these fixture paths.
5. **[Deps · informational] Floating version ranges (`^`)** in all three manifests — but a
   **committed `package-lock.json`** + `npm ci` in CI make installs reproducible, so
   practical risk is low. Dependabot is configured (grouped, majors gated).
